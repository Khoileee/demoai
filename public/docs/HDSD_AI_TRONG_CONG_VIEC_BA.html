Tổng quan 2 công cụ Speech-to-Text

Workspace cung cấp 2 script Python trong thư mục speech_to_text/:

Script Mục đích Input Output
realtime_meeting.py Ghi âm trực tiếp từ microphone và transcribe realtime Âm thanh từ mic máy tính meeting_notes.txt (append liên tục)
test_transcribe.py Transcribe từ file audio có sẵn (.mp3, .wav,...) File audio (ví dụ: recording.mp3) In ra terminal

Bước 2.1 – Cài đặt môi trường Python (1 lần duy nhất)

Yêu cầu: Đã cài Python 3.10+ trên máy. Tải tại https://www.python.org/downloads/

  1. Mở Terminal trong VS Code: nhấn Ctrl + `
  2. Di chuyển vào thư mục speech_to_text:
cd speech_to_text
  1. Tạo môi trường ảo (virtual environment):
python -m venv venv
  1. Kích hoạt môi trường ảo:
.\venv\Scripts\activate
  1. Cài đặt thư viện cần thiết:
pip install faster-whisper sounddevice numpy

Lưu ý: Lần đầu chạy script sẽ tự động download model Whisper (~800MB cho model "turbo"). Cần kết nối internet và thời gian tải khoảng 5–15 phút tùy tốc độ mạng.

Bước 2.2 – Ghi âm trực tiếp cuộc họp (realtime_meeting.py)

Đây là công cụ chính để ghi chép cuộc họp theo thời gian thực.

Cách chạy: 

cd speech_to_text
.\venv\Scripts\python.exe .\realtime_meeting.py

Giao diện điều khiển:

Sau khi model load xong, terminal sẽ hiện:

==================================================
  MEETING TRANSCRIBER
==================================================
  Model   : turbo
  Output  : meeting_notes.txt
==================================================
✅ Model loaded

Controls:
  R → bắt đầu ghi âm
  S → dừng + transcribe
  P → xem nội dung đã lưu
  C → xóa buffer (ghi lại)
  Q → thoát

Các phím điều khiển:

Phím Chức năng Chi tiết
R Bắt đầu ghi âm Micro bắt đầu thu âm, hiển thị 🔴 Recording
S Dừng + Transcribe Dừng ghi, chuyển audio thành text, lưu vào meeting_notes.txt
P Xem notes Hiển thị 20 dòng cuối cùng của file notes
C Xóa buffer Xóa đoạn đang ghi để ghi lại (không xóa file)
Q Thoát Dừng chương trình (hỏi lưu nếu đang ghi)

Quy trình sử dụng trong cuộc họp:

  1. Trước họp: Chạy lệnh khởi động, chờ model load xong
  2. Bắt đầu họp: Nhấn R để bắt đầu ghi
  3. Sau mỗi phần thảo luận: Nhấn S để dừng + transcribe → nhấn R để ghi tiếp
  4. Kết thúc họp: Nhấn S lần cuối → nhấn Q để thoát
  5. Kết quả: Toàn bộ nội dung nằm trong file meeting_notes.txt với timestamp

Hiệu năng tham khảo (CPU, model turbo, int8):

  • 1 phút audio → ~30–40 giây transcribe
  • 5 phút audio → ~2–3 phút transcribe
  • Nên nhấn S sau mỗi 3–5 phút nói để transcript nhanh hơn và chính xác hơn

Ví dụ output trong meeting_notes.txt: 

[2026-03-19 09:30:15]
Đây là các vấn đề liên quan đến đề xuất bổ sung cho VSDF.
Thứ nhất là màn quản lý dashboard. Chuyển trạng thái inactive
của các dashboard công khai và đang được cấu hình trong slide.

[2026-03-19 09:35:42]
Khi mà admin hoặc là người tạo muốn off dash thì sẽ cần off được.
Hiện tại mong muốn là hiển thị list user đang cấu hình dash
trong thông báo lỗi không thể inactive.

Bước 2.3 – Transcribe từ file audio có sẵn (test_transcribe.py)

Dùng khi đã có sẵn file ghi âm (.mp3, .wav, .m4a,...).

Cách chạy:

  1. Đặt file audio vào thư mục speech_to_text/ (ví dụ: recording.mp3)
  2. Mở file test_transcribe.py và sửa tên file audio nếu cần:
segments, info = model.transcribe(
    "recording.mp3",   # ← đổi tên file audio tại đây
    beam_size=2,
    language="vi"
)
  1. Chạy lệnh:
cd speech_to_text
.\venv\Scripts\python.exe .\test_transcribe.py
  1. Kết quả sẽ in trực tiếp ra terminal.

  3. Lovable sẽ mở ra giao diện chat – đây là nơi bạn mô tả giao diện

Bước 3.3 – Viết prompt mô tả UI

Đây là bước quan trọng nhất – bạn mô tả giao diện bằng tiếng Việt hoặc tiếng Anh.

Ví dụ prompt:

Tạo cho tôi một hệ thống quản lý SQL Workflow gồm:
- Sidebar bên trái với các menu: Dashboard, Quản lý kênh, Quản lý template, Quản lý user
- Header phía trên có logo, tên hệ thống, và avatar người dùng
- Trang Dashboard hiển thị 4 card thống kê (Tổng kênh, Tổng template, Tổng user, Tổng workflow)

- Style: clean, professional, dùng màu xanh dương làm màu chính
- Responsive trên cả desktop và mobile

Mẹo viết prompt tốt:

  • Mô tả layout rõ ràng (sidebar trái, header trên, content chính ở giữa)
  • Liệt kê menu và chức năng cụ thể
  • Mô tả bảng dữ liệu với tên các cột
  • Nêu style mong muốn (màu sắc, kiểu dáng)
  • Có thể gửi ảnh chụp màn hình hệ thống mẫu để Lovable tham khảo

  2. Thời gian: khoảng 30 giây – 2 phút
  3. Bạn có thể xem giao diện trực tiếp trong cửa sổ preview

Bước 3.5 – Chỉnh sửa trên Lovable (nếu cần)

Bước 3.6 – Sync code lên GitHub

Phần 4 – Cài đặt môi trường làm việc

Để chạy được project trên máy tính, bạn cần cài 3 phần mềm sau:

Bước 4.1 – Cài đặt Git

Git là gì? – Công cụ giúp máy tính "nói chuyện" được với GitHub (tải code về, đẩy code lên).

  1. Truy cập: https://git-scm.com/downloads
  2. Click "Download for Windows" (hoặc macOS nếu dùng Mac)
  3. Mở file vừa tải về (ví dụ: Git-2.47.1-64-bit.exe)
  4. Cài đặt: cứ nhấn "Next" cho đến khi xong (giữ tất cả tùy chọn mặc định)
  5. Click "Install" → đợi cài → Click "Finish"

Kiểm tra đã cài thành công:

  • Mở Command Prompt (gõ cmd ở thanh tìm kiếm Windows → Enter)
  • Gõ lệnh:
git --version
  • Nếu hiện ra dòng tương tự git version 2.47.1 → thành công

Bước 4.2 – Cài đặt Node.js

Node.js là gì? – Công cụ giúp máy tính chạy được code JavaScript (code mà Lovable đã sinh ra).

  1. Truy cập: https://nodejs.org
  2. Click nút tải bản LTS (Long Term Support – bản ổn định, nút màu xanh bên trái)
    • Ví dụ: 22.14.0 LTS (con số có thể khác tùy thời điểm)
  3. Mở file vừa tải (ví dụ: node-v22.14.0-x64.msi)
  4. Cài đặt: nhấn "Next" cho đến khi xong (giữ mặc định)
  5. Click "Install" → đợi cài → Click "Finish"

Kiểm tra đã cài thành công:

  • Mở Command Prompt (gõ cmd ở thanh tìm kiếm Windows → Enter)
  • Gõ lệnh:
node --version
  • Nếu hiện ra v22.14.0 (hoặc tương tự) → thành công
  • Gõ tiếp:
npm --version
  • Nếu hiện ra số phiên bản (ví dụ: 10.9.2) → thành công

Giải thích: Khi cài Node.js, bạn sẽ được cài kèm npm (Node Package Manager). npm giống như một "App Store" giúp tải các thư viện code cần thiết cho dự án.

Bước 4.3 – Cài đặt Visual Studio Code (VS Code)

VS Code là gì? – Chương trình soạn code miễn phí, và là nơi bạn sẽ dùng AI Agent.

  1. Truy cập: https://code.visualstudio.com
  2. Click "Download for Windows" (hoặc macOS)
  3. Mở file vừa tải (ví dụ: VSCodeUserSetup-x64-1.96.exe)
  4. Cài đặt: nhấn "Next", tích chọn "Add to PATH", tiếp tục "Next" → "Install" → "Finish"

Bước 4.4 – Cài đặt GitHub Copilot trong VS Code

GitHub Copilot là gì? – Trợ lý AI chạy bên trong VS Code, giúp sửa code và sinh tài liệu.

  1. Mở VS Code
  2. Click vào biểu tượng Extensions ở thanh bên trái (hình ô vuông nhỏ, hoặc nhấn Ctrl + Shift + X)
  3. Gõ tìm: "GitHub Copilot"
  4. Click "Install" ở extension "GitHub Copilot" (của GitHub)
  5. Cài luôn "GitHub Copilot Chat" (cũng của GitHub)
  6. VS Code sẽ yêu cầu đăng nhập GitHub → đăng nhập bằng tài khoản GitHub của bạn
  7. Nếu bạn đã mua gói trả phí (Copilot Pro/Business), AI sẽ hoạt động ngay

Lưu ý: Để sử dụng được model Claude Opus 4.6 (model mạnh nhất), bạn cần gói GitHub Copilot Pro hoặc Business.

Phần 5 – Clone repo về máy & mở trong VS Code

Bước 5.1 – Tạo thư mục workspace

Trước hết, bạn cần tạo 1 thư mục chung trên máy để chứa tất cả các repo. Việc này rất quan trọng vì:

  • Agent AI có thể đọc được tất cả các project cùng lúc
  • Bộ skill và cấu hình agent dùng chung cho mọi project
  • Dễ quản lý và tìm kiếm

Cách tạo:

  1. Mở File Explorer (nhấn Windows + E)
  2. Vào ổ đĩa bạn muốn (ví dụ: C:\ hoặc D:\)
  3. Click chuột phải → "New" → "Folder"
  4. Đặt tên thư mục, ví dụ: khoiln1_workspace

Kết quả: Bạn có thư mục C:\khoiln1_workspace (hoặc đường dẫn tương tự)

Bước 5.2 – Mở VS Code và clone repo

  1. Mở VS Code
  2. Ở màn hình chào mừng, click "Clone Git Repository..."
    • Nếu không thấy, nhấn tổ hợp phím Ctrl + Shift + P → gõ "Git: Clone" → chọn "Git: Clone"
  3. Paste link repo GitHub của bạn vào ô nhập, ví dụ:
    https://github.com/Khoileee/sql-wf
  4. Nhấn Enter
  5. VS Code sẽ hỏi "Chọn thư mục đích" → Chọn thư mục workspace bạn đã tạo ở bước 4.1 (ví dụ: C:\khoiln1_workspace)
  6. Click "Select as Repository Destination"
  7. Đợi VS Code tải code về (khoảng 10–30 giây)
  8. VS Code hỏi "Would you like to open the cloned repository?" → Click "Open"

Kết quả: Code đã được tải về tại C:\khoiln1_workspace\sql-wf

Phần 6 – Chạy project trên máy local

Bước 6.1 – Mở Terminal trong VS Code

  1. Trong VS Code, nhấn tổ hợp phím Ctrl + ` (phím backtick, nằm dưới phím Esc)
    • Hoặc vào menu Terminal → New Terminal
  2. Một cửa sổ Terminal sẽ xuất hiện ở phía dưới VS Code

Bước 6.2 – Di chuyển vào thư mục project

Trong Terminal, bạn cần "đi vào" thư mục của project muốn chạy.

Gõ lệnh:

cd sqlwf

Giải thích:

  • cd = "change directory" = "đi vào thư mục"
  • sqlwf = tên thư mục project
  • Nếu muốn quay lại thư mục trước: gõ cd ..

Một số lệnh cd thường dùng:

Lệnh Ý nghĩa
cd sqlwf Đi vào thư mục sqlwf
cd authcenter Đi vào thư mục authcenter
cd .. Quay lại thư mục cha (thư mục bên ngoài)
cd BA-knowledge Đi vào thư mục BA-knowledge

Bước 6.3 – Cài đặt thư viện (npm install)

Gõ lệnh:

npm install

hoặc viết tắt:

npm i

Giải thích đơn giản:

  • Project code cần nhiều "thư viện" bên ngoài để chạy được (giống như xây nhà cần xi măng, sắt, gạch)
  • npm install = "tải tất cả thư viện cần thiết về máy"
  • Thư viện sẽ được lưu trong thư mục node_modules (thư mục này tự sinh ra, không cần quan tâm)
  • Chỉ cần chạy 1 lần cho mỗi project (hoặc khi có thay đổi thư viện)

Đợi khoảng 30 giây – 2 phút. Khi thấy hiện ra dòng tương tự:

added 385 packages in 15s

→ Thành công!

Bước 6.4 – Chạy project (npm run dev)

Gõ lệnh:

npm run dev

Giải thích đơn giản:

  • npm run dev = "chạy project ở chế độ phát triển"
  • Project sẽ khởi động và tạo một "trang web tạm" trên máy của bạn

Khi thấy hiện ra dòng tương tự:

  VITE v6.0.5  ready in 892 ms

  ➜  Local:   http://localhost:8080/
  ➜  Network: use --host to expose

→ Thành công! Project đang chạy.

Bước 6.5 – Xem giao diện

  1. Mở trình duyệt (Chrome, Edge,...)
  2. Gõ vào thanh địa chỉ: http://localhost:8080 (hoặc link hiện trong Terminal)
  3. Nhấn Enter
  4. Giao diện project sẽ hiện ra trên trình duyệt

Lưu ý:

  • localhost = "máy tính của mình" – chỉ bạn xem được, không ai khác thấy
  • Con số 8080 là "cổng" (port) – có thể là 3000, 5173 tùy project
  • Terminal phải luôn mở khi muốn xem giao diện. Nếu tắt Terminal → trang web sẽ mất
  • Muốn dừng: quay lại Terminal, nhấn Ctrl + C

Phần 9 – Đẩy code lên GitHub & cập nhật Lovable

Bước 9.1 – Mở Terminal và vào đúng thư mục

Bước 9.2 – Push code lên GitHub

Gõ lần lượt 3 lệnh sau:

Lệnh 1 – Thêm tất cả thay đổi: 

git add .

Giải thích: "Đánh dấu" tất cả file đã thay đổi, chuẩn bị để lưu. Dấu . có nghĩa là "tất cả file".

Lệnh 2 – Commit (lưu thay đổi kèm mô tả): 

git commit -m "Sửa giao diện sidebar và thêm nút Xuất Excel"

Giải thích: "Lưu lại" các thay đổi với mô tả ngắn gọn trong dấu ngoặc kép. Hãy ghi mô tả rõ ràng để sau này biết đã sửa gì.

Bước 11.1 – Cài đặt OpenClaw

Yêu cầu hệ thống:

  • Windows 10/11 (hoặc macOS/Linux)
  • PowerShell 5.1+
  • Kết nối internet

Lệnh cài đặt:

Mở PowerShell (hoặc Terminal trong VS Code) và chạy:

powershell -c "irm https://openclaw.ai/install.ps1 | iex"

Giải thích: Lệnh này tải và chạy script cài đặt OpenClaw tự động. Sau khi cài xong, lệnh openclaw sẽ có sẵn trong terminal.

Kiểm tra cài đặt thành công: 

openclaw --version

openclaw health

Nếu hiện version number và trạng thái health → cài đặt thành công.

Bước 11.2 – Cấu hình ban đầu (Setup Wizard)

Chạy wizard cấu hình tương tác:

openclaw configure

Wizard sẽ hỏi lần lượt các thông tin:

  1. Workspace path – thư mục làm việc (ví dụ: C:\khoiln1_workspace)
  2. AI Model – chọn model mặc định (xem bước 11.3)
  3. Channel – kênh chat muốn kết nối (Telegram, Discord,...)
  4. Agent defaults – cấu hình mặc định cho agent

Nếu muốn cấu hình lại sau, có thể chạy openclaw configure bất kỳ lúc nào.

Bước 11.3 – Kết nối AI Model

OpenClaw hoạt động như cầu nối giữa kênh chat (Telegram) và các AI model. Cần cấu hình ít nhất 1 model.

Xem danh sách model có sẵn: 

openclaw models scan

Thêm model (ví dụ Claude qua Anthropic API): 

openclaw models add --provider anthropic --api-key sk-ant-xxxxx

Thêm model OpenAI: 

openclaw models add --provider openai --api-key sk-xxxxx

Đặt model mặc định: 

openclaw models set-default --model claude-sonnet-4-20250514

Kiểm tra model đã kết nối: 

openclaw models list

Mẹo: Có thể kết nối nhiều model cùng lúc và chuyển đổi linh hoạt trong chat.

Bước 11.4 – Khởi động Gateway

Gateway là "trung tâm điều phối" của OpenClaw – nhận tin nhắn từ Telegram và chuyển đến AI model.

Khởi động: 

openclaw gateway

Mặc định Gateway chạy trên ws://127.0.0.1:18789.

Khởi động với port khác (nếu cần): 

openclaw gateway --port 19000

Khởi động và tự kill process cũ (nếu bị treo): 

openclaw gateway --force

Quan trọng: Gateway phải luôn chạy trên máy tính để Telegram Bot hoạt động. Khi tắt máy hoặc tắt terminal, bot sẽ ngừng phản hồi.

Bước 11.5 – Tạo Telegram Bot & kết nối

A. Tạo Bot trên Telegram:

  1. Mở Telegram, tìm @BotFather
  2. Gõ /newbot
  3. Đặt tên bot (ví dụ: BA Workspace Bot)
  4. Đặt username (ví dụ: ba_workspace_bot)
  5. BotFather sẽ trả về Bot Token dạng: 7123456789:AAHxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
  6. Copy token này – sẽ dùng ở bước tiếp theo

B. Kết nối Telegram với OpenClaw: 

openclaw channels login --channel telegram --token 7123456789:AAHxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx

C. Kiểm tra kết nối: 

openclaw channels status

Nếu hiện telegram: connected → kết nối thành công.

D. Tìm chat ID của bạn (để pair): 

openclaw directory self --channel telegram

E. Xác minh bằng cách gửi tin nhắn test: 

openclaw message send --channel telegram --target @your_username --message "Hello from OpenClaw!"
